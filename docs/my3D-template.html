my3D — real WebGL2 3D scenes, driven from Clarion

Build and display hardware-accelerated 3D scenes from a Clarion app with a rich object-oriented API — cameras, lights, materials, 20+ mesh primitives, fog, a grid and axes, and genuine Vec3/Mat4 maths — and no JavaScript. One call writes a self-contained .html page and opens it in the browser.

Pure Clarion · no DLL · no COM · no package

Overview top ↑

my3D is one ANSI class, WebGL2Class, plus a template set that wires it into AppGen. You describe a 3D scene by calling methods (add a sphere here, a point light there, set the camera); Show() serialises that scene to a single self-contained HTML file — the scene data plus the verified my3D.engine.js WebGL2 renderer, inlined — and opens it in the default browser. In the page you drag to orbit, mouse-wheel to zoom and press R to reset the camera.

Cameras

Position, look-at target, FOV, near/far clip, and an auto-orbit turntable.

Lighting

Ambient + a directional key light + up to 8 coloured point lights, with distance attenuation.

Materials

Colour, metalness, roughness, opacity, emissive glow, and a wireframe mode — Phong-shaded.

20+ primitives

Box, sphere, cylinder, cone, plane, torus, torus-knot, and the five Platonic solids.

Transforms

Per-mesh position, rotation, scale, and a per-frame auto-spin.

Vec3 / Mat4

Real 3D vector and 4×4 matrix maths that run in Clarion — compute positions yourself.

What it is — and isn't my3D is a scene builder + WebGL2 viewer. It is not a game engine and does not embed a browser control: it writes a page and hands it to the default browser. That keeps it 100 % pure Clarion and makes every generated page a standalone artefact you can share or archive.

Install & register top ↑

Copy these files onto your Clarion redirection path (the app folder, or \clarion12\libsrc\win), stored as ANSI, CRLF:

WebGL2Class.inc — class header (INCLUDE this) WebGL2Class.clw — class implementation (compiled into the app) my3D.engine.js — the WebGL2 renderer (read at RUN time)
Ship my3D.engine.js next to your .EXE The engine is read at run time by SaveHtml()/Show() and inlined into every page the app writes — so each generated .html is self-contained. The class looks for it in the current directory and the app folder (the EngineFile property overrides the name/path). If it is not found, the page instead references <script src="my3D.engine.js">, which still works as long as the JS sits beside the HTML.

Register my3D.tpl in the Clarion IDE (Setup ▸ Template Registry ▸ Register), then either add the my3D - Global extension once at the application level, or just drop the control template (it self-includes the class).

Drag-on control template top ↑

The fastest path. On any window, drop my3D - 3D Scene Viewer button. It places a BUTTON('View 3D Scene...') and wires a fully-configured scene to it: the scene is built when the window opens, and shown when the button is pressed (or automatically on open, if you tick that). It is a MULTI control template, so several independent viewers per window are fine, and it self-includes the class — the global extension is optional.

! The control template generates (essentially) this for you:
Scene1  WebGL2Class                          ! in the window's data

  ! at EVENT:OpenWindow — build the scene from your prompts
  Scene1.SetCamera(7, 6, 11)
  Scene1.AddPointLight(4,3,4, 1,0.4,0.2, 1.2, 18)
  Scene1.AddSphere(0.85, 28)
  ! ... grid, fog, every mesh in the MULTI list ...

  ! at EVENT:Accepted on the button — show it
  Scene1.Show()

Quick start (driving the class directly) top ↑

In a hand-coded program, or from any embed once the class is included:

INCLUDE('WebGL2Class.INC'),ONCE

Scene   WebGL2Class                          ! one object = one 3D scene
M       LONG

  CODE
  Scene.SetCanvas(1000, 640)
  Scene.SetCamera(7, 6, 11);  Scene.LookAt(0, 0.5, 0)
  Scene.SetDirLight(-1,-2,-1.3, 1,0.97,0.9, 1.1)
  Scene.AddPointLight(4,3,4, 1,0.4,0.2, 1.2, 18)      ! a warm point light

  Scene.SetColor(0.20,0.55,0.95)             ! "current material" applies to the next Add*
  Scene.SetMaterial(0.25, 0.35)              ! metalness, roughness
  Scene.SetSpin(0, 0.6, 0)                   ! gentle Y spin
  M = Scene.AddCube(1.4)
  Scene.SetPos(M, -2, 0.8, 0)

  Scene.SetColor(0.20,0.85,0.85);  Scene.SetMaterial(0.7, 0.2)
  Scene.SetPos(Scene.AddTorusKnot(0.7,0.2,2,3), 2, 0.9, 0)

  Scene.Show()                               ! writes my3D_preview.html and launches the browser
The "current material" model Colour, metalness, roughness, opacity, emissive and spin are state on the object. Set them, then call an Add… — the new mesh captures the current values. Change them and add again. SetColorCl() takes a Clarion COLOR instead of three reals.

Scene model & axes top ↑

  • Coordinate system: right-handed. +X right, +Y up, +Z toward the camera. The ground plane and grid lie on Y = 0; the red/green/blue axis helper marks +X/+Y/+Z.
  • Units are arbitrary world units — keep meshes roughly in the −10..10 range and the camera a similar distance out.
  • Rotations (SetRot, spin) are in radians. Use Deg2Rad() if you think in degrees.
  • Mesh indices are 1-based, in creation order. Every Add… returns the new index; MeshCount() returns the last one. Capacity is 256 meshes, 8 point lights.

API · Lifecycle top ↑

Construct() ! sets professional defaults, empty scene Reset() ! back to defaults AND empty the scene ClearScene() ! drop all meshes + point lights, keep settings

Call Reset() at the start of each scene build (the fixtures and the control template both do). ClearScene() keeps your camera/lights/look and just clears geometry.

Composite models — the "special meshes"

Ten ready-made objects, each assembled from many primitives, added with a single call. The first three parameters are the ground origin the model sits on; the fourth is a uniform scale. Each returns the index of the model's first mesh. In the control template these appear in the Shape dropdown as Car (model), House (model), … — pick one and set its Position and Uniform scale (the colour/material prompts don't apply, since each model carries its own palette).

AddCar(x,y,z,scale) AddAirplane(x,y,z,scale) AddRocket(x,y,z,scale) AddWindTurbine(x,y,z,scale) AddRobot(x,y,z,scale) AddTableSet(x,y,z,scale) AddHouse(x,y,z,scale) AddFoundation(x,y,z,scale) AddSkyscraper(x,y,z,scale) AddTrees(x,y,z,scale) 
! build a little town - one call per object
Scene.AddHouse(-6, 0, 4, 0.9)
Scene.AddCar(-5, 0, 8, 0.9)
Scene.AddSkyscraper(7, 0, -6, 0.7)
Scene.AddAirplane(2, 6, -2, 0.7)    ! y=6 -> flying overhead
Roll your own These models are just method bodies that call the primitive Add… functions and then PlaceModel to offset+scale the batch. My3DModels.clw shows each one built from scratch — copy the pattern to compose your own composite meshes.

API · Per-mesh transforms top ↑

SetPos(LONG idx, REAL x, REAL y, REAL z) SetRot(LONG idx, REAL x, REAL y, REAL z) ! radians SetScale(LONG idx, REAL sx, REAL sy, REAL sz) SetUniformScale(LONG idx, REAL s) SetMeshColor(LONG idx, REAL r, REAL g, REAL b) SpinMesh(LONG idx, REAL sx, REAL sy, REAL sz) ! per-frame, rad/s MeshCount(),LONG ! index of the last mesh

All index methods are no-ops if idx is out of range, so SetPos(Scene.AddCube(1), …) is always safe.

API · Output top ↑

SaveHtml(STRING filename),BYTE ! write the page; 1 = ok Show(),BYTE ! save to my3D_preview.html and launch the browser ShowFile(STRING filename),BYTE ! save to your path and launch it

After any of these, LastFile holds the absolute path written. Show() uses the current directory; ShowFile() lets you choose. The browser is launched with RUN('rundll32.exe url.dll,FileProtocolHandler ' & path).

Embedded display — dock the 3D inside a Clarion window

Instead of opening a browser, ShowEmbedded renders the scene inside a Clarion window. It launches a borderless Edge --app window off-screen and re-parents it into the host with the Win32 SetParent. Edge runs in its own process, so Chromium's message pump never re-enters Clarion's event loop — it's stable, and needs no DLL, no import lib, just user32 (which every Clarion app already links) plus Microsoft Edge (Windows 10/11) and my3D.engine.js beside the .exe.

ShowEmbedded(ULONG hostHwnd),BYTE ! dock into hostHwnd (0 = the active Clarion window) EmbedFit(),BYTE ! re-fit the view to the host (call on EVENT:Sized) EmbedSetBounds(x,y,w,h),BYTE ! place the view at an explicit pixel rect EmbedReady(),BYTE EmbedClose() ! is it docked yet / close it (call on EVENT:CloseWindow) 
! a window whose whole client area is the 3D view
  ACCEPT
    CASE EVENT()
    OF EVENT:OpenWindow ;  Scene.ShowEmbedded(0{PROP:Handle})   ! dock once the window is sized
    OF EVENT:Sized      ;  Scene.EmbedFit()
    OF EVENT:CloseWindow;  Scene.EmbedClose()
    END
  END
Pick it in the template The control template's Show in dropdown offers External browser (default) or Embedded; choosing Embedded wires the calls above for you. The docked view fills the host window, so use it on a window dedicated to the 3D. Edge shows a slim app title-bar at the top of the docked view. Set the EdgeExe property if Edge isn't at the default path. Working example: examples/my3D/My3DEmbedded.clw.

How it works top ↑

  1. Your method calls fill arrays in the object (meshes, lights) and scalar fields (camera, fog…).
  2. SaveHtml() streams an HTML page via the ASCII file driver: the head/canvas, then a window.SCENE = {…} block built from those fields, then the contents of my3D.engine.js inlined verbatim.
  3. The engine — a compact data-driven WebGL2 renderer (mat4/vec3, GLSL shaders, primitive geometry builders, Phong shading with the directional + point lights + fog, an orbit camera and a requestAnimationFrame loop) — reads window.SCENE and draws it.
  4. Show() then launches the page in the default browser.
Numbers & colours The class emits compact, locale-safe JS numbers (no grouping commas, signs preserved) and converts Clarion 0xBBGGRR colours to 0..1 RGB. Geometry is generated in the engine, so the Clarion side only ships a small scene description regardless of mesh complexity.

Scope & limitations top ↑

  • Needs a WebGL2-capable browser (every current Edge/Chrome/Firefox). The page reports a clear error if WebGL2 is unavailable.
  • External mode opens a separate browser window (zero dependencies, output shareable). Embedded mode docks the 3D inside a Clarion window (needs Microsoft Edge; see above).
  • No texture images, shadows, or model-file import — this is primitive geometry with solid/Phong materials. The gradient background renders as its dominant clear colour behind the fog.
  • Caps: 256 meshes, 8 point lights per scene.
