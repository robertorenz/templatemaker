myGaugePlus

Beautiful, antialiased analog gauges and dials for Clarion — smooth arcs with rounded caps, a glossy radial-gradient face, an antialiased needle and crisp text. The pretty, GDI+-rendered sibling of the native-graphics myGauge set.

No external dependency Windows · gdiplus.dll v1.0
top ↑

Overview

Clarion's native ARC/ELLIPSE/LINE/POLYGON primitives have no antialiasing, so circular gauges drawn with them look jagged. myGaugePlus draws every pixel with GDI+ instead, into a 32-bit ARGB bitmap, then shows the result as a PNG in an IMAGE control.

The look it gives you out of the box:

  • Glossy face — a radial gradient from a light centre to a soft grey rim, with an optional gloss highlight sweep.
  • Smooth arcs — the track, the coloured value fill and the warning zones are all antialiased with rounded caps.
  • Crisp text — antialiased tick labels, a large value readout and title/units, in any font.
  • Animation — an eased needle sweep driven by the window timer.

It carries no redistributable: gdiplus.dll ships with every Windows since XP, and the bridge to it is a small C file compiled by Clarion's own compiler.

Which gauge template should I use? Use myGaugePlus when looks matter (dashboards, KPI tiles, kiosks). Use the original myGauge when you need report-band gauges or want a pure native-graphics draw with zero PNG/temp-file traffic — myGaugePlus is window-only.

How it works

Three layers, each shipped beside the template:

FileRole
gpcanvas.cA tiny C shim. It binds the GDI+ flat C API at runtime (LoadLibrary("gdiplus.dll") + GetProcAddress) and exposes a simple immediate-mode canvas. Clarion compiles it automatically — see below.
AaCanvasClassA reusable antialiased canvas wrapper: BeginCanvas, Arc, Line, FillCircle, FillCircleGrad, Polygon, Text, SavePng. Useful for any drawing, not just gauges.
GaugePlusClassThe gauge itself, built on the canvas. Same API shape as the native GaugeClass (SetRange/Preset/AddZone/ SetValue/AnimateTo/Draw).

GaugePlusClass.Draw(window, ?image) reads the image control's pixel size, renders the gauge to a PNG in %TEMP%, and points the control at it (?image{PROP:Text}=file). Because the art is now a normal image, Clarion owns all of its repainting and resizing for free. Two temp filenames are alternated each frame so animation reliably reloads.

Zero project setup gpcanvas.c is compiled by a PRAGMA('compile(gpcanvas.c)') inside AaCanvasClass.clw, and the classes' LINK attributes pull everything into the build. You only add the template — no manual project edits.

Install & required files

Copy these five files (shipped beside myGaugePlus.tpl) to a folder on your Clarion redirection path — your app folder or \clarion12\libsrc\win. Save them all as ANSI (not UTF-8):

  • GaugePlusClass.inc  ·  GaugePlusClass.clw
  • AaCanvasClass.inc  ·  AaCanvasClass.clw
  • gpcanvas.c

Register myGaugePlus.tpl in the IDE (Setup ▸ Template Registry ▸ Register), then add the templates to your app. You only ever INCLUDE GaugePlusClass.INC — it pulls in AaCanvasClass.INC for you.

Templates

1 · myGaugePlusGlobal — global extension

Add it once, at the application (global) level. It emits INCLUDE('GaugePlusClass.INC'),ONCE so the classes are available everywhere. Required by the window extension (not needed if you only use the control template, which is self-contained).

2 · myGaugePlus — window procedure extension

Add it once per gauge on a window procedure. You name the gauge object and pick an existing IMAGE control to draw into. Multiple gauges per window are fine — each instance gets its own object. The wiring lives in TakeWindowEvent at PRIORITY(2000): it configures the object on EVENT:OpenWindow, repaints on a private redraw event and on EVENT:Sized, and (optionally) eases the needle on EVENT:Timer.

A per-instance Refresh:Object ROUTINE is generated; call DO Refresh:Gauge1 after a bound value changes when you are not animating.

3 · myGaugePlusControl — control template

The easy path: drag it onto a window and it drops a ready-made IMAGE and wires the gauge to it in one go. WINDOW, MULTI — drop as many as you like. It is fully self-contained (it INCLUDEs the class itself), so it does not require the global extension.

GaugePlusClass — API

The object is declared in your procedure's data; call it from any embed.

Methods

MethodDescription
SetRange(min, max)Set the value range.
Preset(style)Apply a shape preset: 1=180°, 2=270°, 3=360°, 4=90°, 5=45°.
SetSpan(start, sweep)Custom geometry — start angle and sweep in degrees (math convention: CCW from 3 o'clock; negative sweep = clockwise).
SetValue(v)Set the needle value instantly (clamped to range).
AnimateTo(v)Set an animation target; the needle eases there on each AnimStep.
AnimStep(), BYTEEase one step toward the target; returns 1 if it moved (so the caller repaints).
AddZone(from, to, color)Add a coloured warning band over a value range (max 16). color is a Clarion COLOR.
ClearZones()Remove all zones.
Draw(window, ?image)Render the gauge and show it in the IMAGE control.

Hand-coded use

You don't need the template — the classes work standalone:

! one gauge object in your procedure's data
Speedo   GaugePlusClass

  CODE
  Speedo.SetRange(0, 220)
  Speedo.Preset(GaugeP:Arc270)
  Speedo.MajorTicks = 11; Speedo.MinorTicks = 1
  Speedo.AddZone(170, 220, 02020E0h)   ! red zone (Clarion COLOR)
  Speedo.Title = 'SPEED'; Speedo.Units = 'km/h'
  Speedo.SetValue(146)
  Speedo.Draw(Window, ?Img)             ! render + show
  ! ... later, for a smooth sweep on a window timer:
  Speedo.AnimateTo(80)
  IF Speedo.AnimStep() THEN Speedo.Draw(Window, ?Img).

See the buildable examples in examples/myGaugePlus/: AaTest.clw (canvas self-test), GaugePlusTest.clw (headless gauge render) and GaugePlusDemo.clw (a live, animated window).

Notes & gotchas

  • Transparency. By default the canvas background is transparent, so a round gauge composites cleanly onto any window colour. Set BackColor if you want a solid backdrop.
  • Window-only. Because it renders via PNG + window pixel sizing, myGaugePlus draws on windows, not report bands. For report gauges use the native myGauge.
  • Temp files. Each gauge writes two small PNGs to %TEMP% (gplus_feq_a/_b.png), alternated so animation reloads reliably.
  • ANSI source. Like all Clarion include/source files here, keep the five shipped files in ANSI with CRLF line endings.
Under the hood The C shim hand-declares the few Win32 functions it needs (Clarion's bundled compiler has no Windows SDK headers) and spells the stdcall convention pascal. None of this concerns the template user — it just compiles.
