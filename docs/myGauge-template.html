myGauge

A configurable analog gauge / dial for Clarion windows and reports, drawn entirely with native graphics primitives (ARC, ELLIPSE, LINE, POLYGON, SHOW) into an IMAGE control. No images, no temp files, no external dependencies — pure Clarion.

ABC & Clarion · 45°–360° spans · zones · ticks · smooth animation · self-contained

Overview #

myGauge turns any number — a KPI, a temperature, a percent-complete, a battery level — into a polished speedometer-style dial painted directly onto a window or a report band, computed and drawn on the local machine.

It is a sibling of myPie and myQRDraw: where those draw charts and codes, myGauge draws an instrument. A single GaugeClass object holds the configuration (range, span, colors, ticks, zones) and renders itself. The myGauge template puts a full prompt UI in front of the class so you can drop a gauge on a window without writing a line of graphics code — and still reach the object from any embed for run-time control.

Install & register #

  1. Copy templates/myGauge/GaugeClass.inc and GaugeClass.clw to a folder on the Clarion redirection path (your app folder, or \clarion12\libsrc\win). They must be stored in ANSI, not UTF-8.
  2. Copy templates/myGauge/myGauge.tpl into your template folder.
  3. In the IDE: Setup ▸ Template Registry ▸ Register, pick myGauge.tpl.
  4. The chain myGauge registers one control template (the drag-on gauge) and three extensions: the global include, the window drawer, and the report drawer.
Already on the redirection path If you installed the template set into \clarion12, the class files are already in libsrc\win and the .tpl in accessory\template\win — just register the chain.

Drag-on control template #

The fastest way to add a gauge: open the window in the Window Designer, find myGauge - Analog Gauge in the control template list, and drop it on the window.

  1. In the Window Designer, open the control template toolbox (the same place Browse boxes and Calendar buttons live) and pick myGauge - Analog Gauge (drag onto a window).
  2. Click where you want it. An IMAGE control appears, already attached to a gauge object — resize it like any control; the dial fills it.
  3. Open the control's Actions (template properties) and set the Range, style, value, colors, ticks, zones, and animation — the same options the extension offers.
  4. Compile and run. Nothing else to wire.
Self-contained — no global extension needed The control template pulls in GaugeClass itself (an INCLUDE with ONCE), declares its own object, and draws on open/resize. Drop several on one window; each gets its own object and its own auto-named IMAGE. You can still add the myGaugeGlobal extension too — the ONCE guard means the class is never included twice.

Everything in the rest of this manual — shapes, zones, animation, the run-time API — applies identically; the control template just creates the IMAGE for you instead of you placing one and pointing the extension at it.

Quick start (extension) #

Prefer to use your own IMAGE control, or wiring an existing window? Use the extension path:

  1. On the global Extensions for the application, add myGauge - Global (once).
  2. Put an IMAGE control on your window where the dial should appear (give it a sensible size — the gauge fills it).
  3. On that window's procedure Extensions, add myGauge - Draw a gauge on this window.
  4. Set the Image control to draw into, the Range (min/max), pick a style, and either type an initial value or tick Value is a variable and name the field.
  5. Compile and run. The gauge paints on EVENT:OpenWindow and repaints when the window is resized.

That is the whole no-code path. Everything below is for going further.

Gauge shapes #

The style drop-list picks a preset arc. Each preset sets the start angle, sweep, and where the pivot sits inside the control so the dial is well-framed:

270° Speedometer

The default. Classic dial, opens at the bottom.

180° Semicircle

A half-dial sitting on its diameter.

360° Full dial

A complete circle — clocks, compasses.

90° Quarter

A corner gauge; pivot tucked low-left.

45° Narrow arc

A shallow band for tight layouts.

Custom

Your own start angle + sweep (deg).

Angles use the math convention: 0° is 3 o'clock and positive is counter-clockwise. A negative sweep goes clockwise — so the speedometer is start 225, sweep -270 (from upper-left, clockwise round the bottom to upper-right).

Changing it at run time #

The object lives in the procedure's data, named as you set in the prompts. Call it from any embed (a button, a TakeAction, after a browse refresh):

! instant update — set the value and repaint now
Gauge1.SetValue(PRE:Score)
Gauge1.Draw(?GaugeImage)

! or, if the value sits in a field, just call the generated refresh routine
DO Refresh:Gauge1

! retune it live
Gauge1.SetRange(0, 250)
Gauge1.TrackColor = COLOR:Silver
Gauge1.Draw(?GaugeImage)
Refresh:<Object> routine For every window gauge the template generates a Refresh:ObjectName ROUTINE. It re-reads the value field (if any) and redraws — the simplest way to update a non-animated gauge after the underlying field changes.

Animation #

Tick Animate the needle and the gauge eases toward new values instead of snapping. It works through the window timer:

  • On open, the template sets 0{PROP:Timer} to your interval.
  • Each EVENT:Timer the gauge calls AnimateTo() (if the value is a field) then AnimStep(), which moves the needle one step toward the target and repaints. AnimStep() returns 1 while still moving, 0 when settled.
  • Step per tick is a percent of the full range, so the sweep speed is independent of the range size.
One timer per window A window has a single timer. If another template or your own code also uses PROP:Timer, share it: call the gauge's AnimStep() from your existing EVENT:Timer handler instead of enabling the template's animation.
! drive animation from your own timer
OF EVENT:Timer
  Gauge1.AnimateTo(PRE:Live)
  Gauge1.AnimStep()

Colored zones #

Zones are colored bands over a value range, drawn on top of the empty track and under the ticks. Add them in the Zones tab, or at run time:

Gauge1.ClearZones()
Gauge1.AddZone(0,  60,  COLOR:Green)
Gauge1.AddZone(60, 85,  0012A8E8H)   ! amber
Gauge1.AddZone(85, 100, COLOR:Red)
Gauge1.Draw(?GaugeImage)

Up to 16 zones are stored per gauge. Each zone is just an ArcBand — a thick ARC — so they cost almost nothing to draw.

On a report #

Add myGauge - Draw a gauge on this REPORT to a report procedure. Put an IMAGE control in a detail (or group) band and select it in the Image control picker — the list is populated from the report's own IMAGE controls.

The template draws the gauge for each record at %BeforePrint, reading your value field, with the report band as the target:

  Gauge.SetValue(PRE:Rating)
  SETTARGET(Report)        ! draw into the report band, not the screen
  Gauge.Paint(?RptGaugeImage)
  SETTARGET()
Paint vs Draw Draw() retargets the named IMAGE control and clears it — right for windows. Paint() draws into whatever target is current — right for reports, where the template has already called SETTARGET(Report).

Architecture #

templates/myGauge/ ├─ GaugeClass.inc the class: properties, equates, method prototypes ├─ GaugeClass.clw the implementation (geometry + drawing) └─ myGauge.tpl three extensions: global include, window, report

The class is self-contained the same way myQRDraw's is: a CLASS,TYPE,MODULE,LINK whose .clw opens with a bare MEMBER and a module-level MAP, so the Clarion graphics built-ins in BUILTINS.CLW resolve. Each gauge on a window is its own local object, so multiple gauges per window or report just work — each template instance declares and drives its own instance.

Geometry & drawing #

Everything is derived from the control's size at draw time (GETPOSITION), so the gauge scales with its IMAGE control and survives a resize.

  • Value → angle. angle = StartAngle + ((v − Min) / (Max − Min)) × Sweep. Angles are kept un-normalized (they may be negative or > 360) so they stay monotonic along the sweep and never trip over the 0/360 boundary.
  • Screen Y. Y grows downward, so a point at math angle θ is (cx + r·cos θ, cy − r·sin θ). Native SIN/COS do the trig — no lookup table.
  • Bands (track + zones) are a thick-pen ARC; ticks are LINEs; labels / title / readout are SHOW; the triangle needle is a 3-point POLYGON (a thick LINE for the line style); the hub is a filled ELLIPSE.

Troubleshooting #

SymptomCause / fix
Nothing drawsThe Image control prompt isn't set, or the IMAGE has zero size. Give the control a real width/height.
Template won't registerA literal % in an emitted line must be %%; a literal < in a Clarion string must be <<. Re-save the class files as ANSI, not UTF-8.
"Unresolved external" on a built-inThe .clw must open with a bare MEMBER and a module-level MAP so BUILTINS.CLW resolves. Don't remove them.
Needle doesn't moveAnimation needs the window timer. Either tick Animate, or call AnimStep() from your own EVENT:Timer.
Gauge clipped on resizeLower RadiusPct, or pick a preset whose pivot suits the control's aspect ratio.
Report gauge blankMake sure the IMAGE is in a band that prints and the draw is wrapped in SETTARGET(Report) … SETTARGET().
