Clarion Template Designer

A visual designer for the prompt UI of Clarion templates. Open a .tpl, see every control where it really sits, then drag, style, re-order and rename — and save changes back into the template with surgical precision.

User Manual · version 2.3

Welcome ↑ top

The designer reads a Clarion template's #SHEET / #TAB / #PROMPT structure and lays the controls out on a canvas at their real AT(x,y,w,h) positions — icons render as the actual PNGs. You edit visually; when you save, the designer rewrites only what you changed (positions, styles, added/removed/relocated controls) and leaves every other byte of the template untouched.

Who this is for Anyone who maintains Clarion .tpl/.tpw templates and would rather lay out a prompt window by eye than count dialog units by hand.

Opening a template ↑ top

  1. Choose File ▸ Open .tpl… and pick a template file.
  2. The designer follows any #INCLUDE('…tpw') files automatically, so the whole template set loads together.
  3. The status bar reports how many editable parts, components and files were found.

Parts & tabs ↑ top

A template can contain many components — one might be global, another an extension, another a procedure. Each component that has a prompt sheet is a selectable Part.

  • Part selector — switch between the components in the template set (across all files).
  • Tab selector — switch between the #TABs within the current part's sheet.
Tip Switching parts also reloads the Source panel to the file that part lives in, so the source view always matches what you see.

The workspace ↑ top

  • Canvas — the centre, showing the current tab's controls over a dialog-unit grid with rulers.
  • Properties — the control's text, position, style, symbol and usages.
  • Source — the live template text in a colour-coded editor with a minimap.
  • Command bars — quick controls for preview, style and inserting controls.

A fixed icon command bar sits under the menu with one-click Open, Recent ▾, Save, Preview changes, Undo/Redo, Copy/Paste, Check problems, Find and Preview. Templates you open are remembered in the Recent dropdown and under File ▸ Open Recent — just opening a file adds it (no save needed).

Every panel is dockable — drag a panel's header to re-dock it to any edge, or tear it off into a floating window. View ▸ Reset panel layout puts everything back.

Selecting controls ↑ top

  • Click a control to select it.
  • Ctrl+click toggles a control in/out of the selection.
  • Shift+click adds a control to the selection.

A single selection shows resize handles and the full Properties pad. When you select several, their lines are also highlighted in the Source panel.

Move, resize & nudge ↑ top

  • Drag a control to move it; drag a handle to resize.
  • Arrow keys nudge the selection 1 unit; Shift+arrow nudges 5 units.
  • Snapping to the grid and to guides can be toggled in Preferences.
  • The zoom control scales the canvas without changing any coordinates.
Need everything draggable? Some controls have no explicit position yet. Edit ▸ Add AT to all controls gives each positionable control an AT(x,y,w,h) from the current layout so it can be dragged. Prompts that Clarion auto-builds (dropdowns, “…” buttons) are skipped on purpose.

Align & distribute ↑ top

Select two or more controls (Ctrl/Shift-click), then use Arrange ▸ Align, Same size and Distribute — or right-click the selection for the same menu.

  • Align — left / centre / right and top / middle / bottom, relative to the selection's bounds.
  • Same size — match width, height or both to the largest in the selection.
  • Distribute — spread three or more controls evenly (the end ones stay put).
Tip Right-clicking a control that isn't in the current selection selects just that one; right-click within a multi-selection to act on the whole group.

Adding controls ↑ top

Use the Insert menu or the Add command bar to drop in a Label, String, Number, Spin, Check box, Image or Group box.

Adding a tab

Insert ▸ New Tab… (or the + Tab button on the Add bar) adds a whole new #TAB to the current part's sheet. You're asked for the tab's caption; the new tab is inserted after the existing ones and selected, ready for you to add controls to it. It's written to the template on Save.

Reordering tabs

In Flow preview, drag a tab's header onto another tab to move it to that position. A red caret follows your cursor along the tab strip and snaps to the edge where the tab will land — drop on the left half of a tab to land before it, the right half to land after. The whole #TAB…#ENDTAB block — and everything in it — moves with it; save to write the new order.

Right-click a tab's header to Rename it (the caption is rewritten in place) or Delete it (the whole tab and its controls are removed on save — you're warned if its symbols are used elsewhere, and a sheet always keeps at least one tab).

  • New controls appear selected and ready to position.
  • In Flow preview you can drag a control straight from the Add bar onto the exact spot (line) where you want it inserted.
  • For images, use the … button to browse for a file, then ↻ Refresh to re-render after typing or changing the file name.
Name it as you add it A new control gets an auto field name. With it selected, type a name in the Properties SYMBOL box and click Set name. See Symbols & rename.

Copy, paste & duplicate ↑ top

  • Ctrl+C / Ctrl+X copy or cut the selected control(s).
  • Ctrl+V pastes into the current tab — or into the selected group box — nudged slightly so the copy isn't exactly on top.
  • Ctrl+D duplicates in place.

A group box pastes with all its children, and every pasted #PROMPT gets a fresh unique %symbol so a paste never silently duplicates a field. Also on the Edit menu and the right-click menu.

Snippets ↑ top

Insert ▸ Snippets drops a ready-made pattern into the current tab (or selected box) in one click — a group box with three fields, an option group with three radios, two columns of fields, or two check boxes. Each gets fresh %symbols; tweak captions, positions and types afterwards like any other control.

Prompt type & attributes ↑ top

When you select a #PROMPT you add, the Properties pad shows a PROMPT TYPE section:

  • Type — pick a common picture/type (@s255, @n8, CHECK, SPIN(…), DROP('A|B'), OPTION, PROCEDURE…) or type your own token.
  • Required (REQ) — mark the entry as must-fill.
  • Default — the literal placed inside DEFAULT(…), e.g. 'AJE', 5 or %Symbol.

These are editable for controls you add; for existing controls they're shown read-only (edit them in the source). See also adding a tab for renaming and deleting tabs.

Properties & style ↑ top

The Properties pad shows, for the selected control:

  • Text / label / file — the control's caption or image file (editable for controls you add).
  • X / Y / W / H — position and size in dialog units.
  • Style — font, size, bold and colour. Changes are reflected on the canvas and written back as the control's PROP(…) on save.
  • Source — the control's exact template line(s), read-only.

The Style command bar offers the same font / size / colour changes for fast edits.

Symbols & rename ↑ top

Most prompts are bound to a %Symbol — a template field name. Selecting such a control reveals a SYMBOL section in the Properties pad:

  • Uses list — every line, across every file in the template set, where the symbol appears. The control's own declaration is marked with ●; the rest are the generation/logic code that reads it. Click any entry to jump the source editor straight to that line.
  • Rename — type a new name and click Rename (or press Enter). The symbol is renamed everywhere at once — the prompt and every reference — so the field stays joined. It validates the name and refuses to merge into a symbol that already exists.
  • Set name — for a newly added control, the same box names its field.
Pending vs saved The Uses list reflects the file on disk. In Live (pending) source mode, a jump can land a line or two off until you save. Rename is fully undoable.

Z-order & groups ↑ top

When controls overlap, use the Arrange menu — Bring to Front, Bring Forward, Send Backward, Send to Back — to control which sits on top.

Group boxes (#BOXED) contain other controls. Drop a control inside a group box to make it a child; drag it out to remove it. Moving the box carries its contents along.

Deleting safely ↑ top

Delete a control with the Delete key or right-click ▸ Delete.

Reference check If the control's %symbol is used elsewhere in the template (usually the generation/logic code), the designer warns you and lists where — deleting the control removes where the symbol is set and can break code generation. You decide whether to proceed.

Rulers & guides ↑ top

  • Drag from the top ruler to create a horizontal guide; from the left ruler for a vertical guide.
  • Hold Ctrl while dragging to snap the guide to ruler segments.
  • Drag a guide back onto a ruler to remove it, or use Guides ▸ Clear guides.
  • Controls snap to guides when Snap to guides is on.

Outline & find ↑ top

The Outline panel (left) shows the whole structure of the current part — #TAB → #BOXED → controls — as a tree. Click a node to select that control (switching tabs as needed); the tree also tracks whatever you select on the canvas.

The find box at the top filters the tree by control caption or %symbol; press Enter to jump to the first match. Toggle the panel from View ▸ Outline panel.

Problems panel ↑ top

The Problems panel (tabbed with Source) checks the current part and lists issues — it runs when you open a template, when you save, and via Edit ▸ Check for problems or the panel's ↻ Re-check button. Click a problem to select the offending control and highlight its source line.

It flags:

  • duplicate %symbols within the part;
  • negative or off-canvas positions, and overlapping controls;
  • explicit AT on prompts Clarion auto-builds (which can hide the generated part);
  • declared symbols never referenced elsewhere;
  • unbalanced #SHEET/#TAB/#BOXED/#BUTTON/#ENABLE blocks in any file.

Flow preview ↑ top

Toggle View ▸ Flow preview to see the prompt rendered as real, flowed controls — the way AppGen lays them out — instead of the absolute-position canvas. The preview is also a live editor:

  • Drag a control to a new line to re-order it; the underlying source moves with it.
  • Drag a control into a group box, or onto another tab header, to reparent it.
  • Drag from the Add bar onto the preview to insert a new control where you drop it.
  • Edit a control's text, font and colour directly.

Preview width

Switch the canvas width to match your Clarion version: 480 for Clarion 10 and earlier, 960 for Clarion 11.0 / 11.1 / 12.

Saved vs unsaved

Preview unsaved work (on) shows your in-progress, interactive model. Turn it off to preview the saved file exactly as it is on disk (read-only).

Source panel ↑ top

  • Colour-coded Clarion template syntax with a minimap for quick navigation.
  • Selecting controls highlights their lines; clicking a line scrolls to the matching control.
  • Live (pending) shows the source as it would be saved, reflecting all unsaved edits.
  • Apply writes your hand-edits in the editor back to the file and re-parses; Revert discards them.
  • Ctrl+F opens find / replace in the editor.
  • Autocomplete — type % for a list of every symbol in the source, or # for the common template directives.

Saving & diff ↑ top

File ▸ Preview changes… shows a colour-coded, per-file diff of exactly what a save would write versus the files on disk (added lines green, removed red) — with a Save now button right in the dialog. Use it to make every save predictable.

File ▸ Save (Ctrl+S) writes your changes. The writer is conservative: it rewrites only the AT() of moved/resized controls, updates PROP(…) for style edits, drops deleted lines, relocates reparented controls, emits added ones, and applies symbol renames across files. Everything else is preserved byte-for-byte. Only files that actually changed are written — untouched .tpw includes are left alone.

Preferences & layout ↑ top

The Preferences menu toggles Show grid, Snap to grid, Snap to guides and Show minimap — all of which persist between sessions, along with your panel layout. Re-dock or float any panel; View ▸ Reset panel layout restores the default arrangement.
