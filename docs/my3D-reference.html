WebGL2Class — Complete Programmer's Reference

Every property and every method of the my3D 3D-scene class, each with example code. The class builds a scene through a fluent API and emits a self-contained WebGL2 page that it opens in the browser or docks inside a Clarion window.

Pure Clarion · no DLL · no COM · companion file my3D.engine.js

Overview top ↑

Declare one object per scene. Configure it (camera, lights, materials, meshes), then Show() (browser) or ShowEmbedded() (in a Clarion window).

INCLUDE('WebGL2Class.INC'),ONCE      ! once, globally (or via the my3D template)

Scene   WebGL2Class                  ! Construct() runs automatically -> sensible defaults

  CODE
  Scene.SetCamera(7, 6, 11)
  Scene.SetDirLight(-1,-2,-1.3, 1,0.97,0.9, 1.1)
  Scene.AddPointLight(4,3,4, 1,0.4,0.2, 1.2, 18)
  Scene.SetColor(0.20,0.55,0.95);  Scene.SetSpin(0,0.6,0)
  Scene.AddCube(1.4)
  Scene.Show()                       ! writes my3D_preview.html and launches the browser
Ship two files at run time Compile WebGL2Class.inc + WebGL2Class.clw into your app, and ship my3D.engine.js beside the .exe — it is read at run time and inlined into every page, so the renderer and the class must be the same version. (If a new flag like a view-option seems ignored, your engine.js is stale.)

Conventions used below

  • All colour channels are REAL 0.0–1.0. Methods ending …Cl take a Clarion COLOR (0xBBGGRR) instead and convert it for you.
  • Rotations/spins are in radians (use Deg2Rad() if you think in degrees).
  • Mesh adders return the new mesh's 1-based index (LONG); they are ,PROC so you may also call them as a statement.

Scene model & units top ↑

  • Right-handed axes: +X right, +Y up, +Z toward the camera. The grid/ground sits on Y=0; the axis helper marks +X (red), +Y (green), +Z (blue).
  • Units are arbitrary world units — keep meshes in roughly −10..10 and the camera a similar distance out.
  • Capacity: 256 meshes (WebGL2:MaxMesh), 8 point lights (WebGL2:MaxLight). Adders past the cap return 0.
  • The "current material" (colour, metalness, roughness, opacity, emissive, spin) is object state: set it, then the next Add… captures it.

Lifecycle top ↑

Construct()

Runs automatically when the object is declared. Sets the engine file (my3D.engine.js), the Edge path, the title-bar inset, then calls Reset().

Scene   WebGL2Class      ! that's it - Construct() already configured defaults
Reset()

Returns every setting to its default and empties the scene. Call it at the start of each scene build (the template and all examples do).

Scene.Reset()
Scene.SetTitle('Frame 2')
! ... build the next scene ...
ClearScene()

Drops all meshes and point lights but keeps camera, lights, colours, grid, etc. Handy for re-drawing a changing dataset without re-configuring the view.

Mesh primitives top ↑

Each returns the new 1-based mesh index. Set the material first; position/rotate after with the transform methods.

Generic adder. pType is one of 'box' 'sphere' 'cylinder' 'cone' 'plane' 'torus' 'torusknot' 'tetra' 'octa' 'icosa' 'dodeca'; the params mean what the matching Add… takes. All the typed adders below call this.

M = Scene.AddMesh('sphere', 0.8, 24, 0,0,0,0)

M = Scene.AddBox(2, 1, 0.5)

Composite models top ↑

Ready-made objects assembled from many primitives in one call. The first three params are the ground origin the model sits on; the fourth is a uniform scale. Each returns the index of the model's first mesh.

AddCar(x,y,z,scale) · AddAirplane(…) · AddRocket(…) · AddWindTurbine(…) · AddRobot(…) , LONG AddTableSet(…) · AddHouse(…) · AddFoundation(…) · AddSkyscraper(…) · AddTrees(x,y,z,scale) , LONG 
! a little town - one call per object
Scene.AddHouse(-6, 0, 4, 0.9)
Scene.AddCar(-5, 0, 8, 0.9)
Scene.AddSkyscraper(7, 0, -6, 0.7)
Scene.AddAirplane(2, 6, -2, 0.7)     ! y=6 -> flying overhead

These are just method bodies that call the primitive Add… functions then PlaceModel to offset+scale the batch — copy the pattern in examples/my3D/My3DModels.clw to build your own.

Embedded display (docked Edge overlay) top ↑

Renders the scene inside a Clarion window by positioning a borderless Microsoft Edge --app window as an owned overlay (a frameless, non-resizable top-level window owned by your window), so it keeps full native mouse + keyboard. Needs Edge (Win10/11) and my3D.engine.js beside the .exe.

ShowEmbedded(ULONG pHostHwnd) , BYTE

Writes the page and shows the overlay over pHostHwnd (pass 0{PROP:Handle}, or 0 to use the active window). Call it on EVENT:OpenWindow so the window is sized first.

SetEmbedControl(SIGNED pFeq)

Confine the overlay to a control's rectangle (an IMAGE/REGION placeholder); 0 = the whole window. Call it before ShowEmbedded.

EmbedFit() , BYTE

Re-fit the overlay to the host/control. Call on EVENT:Sized AND EVENT:Moved so it tracks the window.

EmbedSetBounds(x,y,w,h) , BYTE · EmbedReady() , BYTE · EmbedFocus() · EmbedClose()

EmbedSetBounds places it at explicit screen pixels; EmbedReady is 1 once shown; EmbedFocus hands it the keyboard; EmbedClose closes it (on EVENT:CloseWindow).

! a window whose whole client area is the 3D view (or set a control first)
  Scene.SetEmbedControl(?View)             ! optional: dock into an IMAGE control
  ACCEPT
    CASE EVENT()
    OF EVENT:OpenWindow ;  Scene.ShowEmbedded(0{PROP:Handle})
    OF EVENT:Sized      ;  Scene.EmbedFit()
    OF EVENT:Moved      ;  Scene.EmbedFit()
    OF EVENT:CloseWindow;  Scene.EmbedClose()
    END
  END
Properties you can tune EdgeExe (path to msedge.exe), EmbedInset (px shifted up to hide Edge's title bar; default 33 — bump it at higher display scaling), and DisplayMode (WebGL2:External/WebGL2:Embedded, set for you).
