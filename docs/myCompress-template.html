myCompress

A self-contained compression library for Clarion — DEFLATE, zlib and gzip implemented in pure Clarion. Compress and decompress memory buffers and files, in formats that interoperate with gzip, 7-Zip, browsers and .NET. No DLL, no external library.

RFC 1951/1950/1952 · memory + files · gzip/zlib/raw · one global object · pure Clarion

Overview #

myCompress adds one global object to your application — a CompressClass instance you can reach from any procedure or embed — and through it compress or decompress data, either in memory or on disk, using the standard DEFLATE family of formats.

It is a sibling of myQRDraw and myBarcodeGen: a self-contained ANSI Clarion class that does real work with no outside dependency. The codec is a from-scratch implementation of RFC 1951 (DEFLATE) with the RFC 1950 (zlib) and RFC 1952 (gzip) wrappers, so a .gz it writes opens in any tool, and it reads any .gz / zlib / raw stream those tools produce.

Install & register #

  1. Copy templates/myCompress/CompressClass.inc and CompressClass.clw to a folder on the Clarion redirection path (your app folder, or \clarion12\libsrc\win). They must be stored in ANSI, not UTF-8.
  2. Copy templates/myCompress/myCompress.tpl into your template folder.
  3. In the IDE: Setup ▸ Template Registry ▸ Register, pick myCompress.tpl.
  4. On the application's global Extensions, add myCompress - Global Compressor (once). Set the object name (default Zip), and optional default format and level.
That's the whole setup The class's LINK attribute pulls CompressClass.clw into the build automatically; the ONCE include guard means it is safe even if another template also pulls it in. The object is multi-DLL aware out of the box (defined in the root DLL, imported elsewhere).

Quick start #

From any embed, call the global object (named in the prompts; Zip below):

! ---- memory: compress a string, then get it back ----
loc:raw    STRING(5000)
loc:rawLen LONG
packed     &STRING
n          LONG
back       STRING(5000)
  CODE
  loc:raw    = '…whatever bytes you want to pack…'
  loc:rawLen = 5000                          ! the REAL length — see the binary note
  packed &= NEW STRING(Zip.MaxCompressed(loc:rawLen))
  n = Zip.Compress(loc:raw, loc:rawLen, packed)      ! n = packed byte count, or -1
  IF n > 0
    Zip.Decompress(packed, n, back)                  ! back now equals loc:raw
  END
  DISPOSE(packed)

! ---- files: one call each way ----
  IF Zip.CompressFile('report.txt', 'report.txt.gz')
    ! report.txt.gz now opens in 7-Zip / gunzip / any browser
  END
  IF Zip.DecompressFile('archive.gz', 'archive.out')   ! reads any standard .gz
  END
Always pass the length — never rely on LEN() A Clarion STRING is fixed-length and space-padded; it does not remember how many bytes are "real", and compressed data contains 0s and spaces like any other byte. Every method here is length-explicit: you pass a LONG length in and get a LONG length back. Store that length next to the bytes (e.g. in a separate field, or a 4-byte prefix) — if you let trailing spaces creep in, the data is corrupt.

Compressing memory #

The two memory methods are length-explicit and write into a caller-supplied output buffer:

  • Compress(*STRING pIn, LONG pInLen, *STRING pOut) — returns the compressed byte count, or -1 on error. pOut must be at least Zip.MaxCompressed(pInLen) bytes.
  • Decompress(*STRING pIn, LONG pInLen, *STRING pOut) — returns the original byte count, or -1. pOut must be big enough for the decompressed data (you generally know this from your own format, or store it alongside the compressed blob).
  • MaxCompressed(LONG pInLen) — the safe size to allocate for a compress output buffer (input may slightly expand for incompressible data, plus header/trailer).

Any memory, not just strings. A *STRING parameter accepts a STRING you overlay on anything — a GROUP, a queue record, a BLOB you have loaded — so you can compress arbitrary structures:

MyRec  GROUP
Id       LONG
Name     STRING(60)
…
       END
MyRecS STRING(SIZE(MyRec)),OVER(MyRec)        ! a byte view over the group
  CODE
  n = Zip.Compress(MyRecS, SIZE(MyRec), packed)

Compressing files #

The file methods read the whole source file, run it through the same engine, and write the result — returning 1 on success, 0 on failure (with Zip.ErrText set):

  • CompressFile(STRING pSrc, STRING pDst) — writes pDst in the current Format (gzip by default, so name it .gz).
  • DecompressFile(STRING pSrc, STRING pDst) — auto-detects gzip / zlib / raw and writes the decoded bytes.
Whole-file in memory (this release) Files are loaded fully into memory for v1, which is ideal for documents, reports and exports up to tens of MB. True chunked streaming for multi-GB files is a planned enhancement; until then, size your expectations to available memory.

Formats & interop #

Set Zip.Format before a call (or pick the default in the prompts). The three formats are the standard DEFLATE family:

FormatEquateWrapperReads / writes with
gzipCmp:Gzip (2)10-byte header + CRC32 + ISIZEgzip / gunzip, 7-Zip, browsers, .NET GZipStream, *.gz
zlibCmp:Zlib (1)2-byte header + Adler32zlib, .NET ZLibStream, PNG/HTTP payloads
rawCmp:Raw (0)none — bare DEFLATE bitstream.NET DeflateStream, embedding in your own container

Decompression auto-detects — Decompress / DecompressFile sniff the gzip magic (1F 8B) and the zlib header check, falling back to raw DEFLATE — so you do not have to know which format a blob is in to read it.

Compression ratio The compressor currently uses fixed Huffman coding (plus LZ77 matching), which every decoder reads but which is a little larger than zlib's best (zlib adds dynamic Huffman). Decompression already handles dynamic-Huffman input from any encoder. Dynamic-Huffman output for tighter ratios is the next planned step.

Run-time control #

The object exposes a few public properties; set them any time before a call:

PropertyMeaning
Level0 = stored (no compression, valid container), 1 fast … 9 best. Higher levels search the match chains harder. Default 6.
FormatCmp:Raw / Cmp:Zlib / Cmp:Gzip. Default Cmp:Gzip.
ErrCode / ErrTextAfter any call: 0 = ok, negative = the error (see Error codes); ErrText is the readable companion.
  Zip.Level  = 9                    ! squeeze harder for an archive
  Zip.Format = Cmp:Zlib            ! emit a zlib stream for an HTTP payload
  IF Zip.Compress(raw, rawLen, out) < 0
    Message('Compress failed: ' & Zip.ErrText)
  END
Sanity check at startup IF ~Zip.SelfTest() round-trips a known buffer through all three formats and returns 1 on success — a one-line smoke test you can run once when the program starts.

Architecture #

templates/myCompress/ ├─ CompressClass.inc the class: config, equates, method prototypes ├─ CompressClass.clw the engine (inflate, deflate, containers, checksums) └─ myCompress.tpl one global extension (adds the shared object) designer/CompressCore/ the .NET golden-vector oracle (validation)

The class is self-contained the same way myQRDraw's encoder is: a CLASS,TYPE,MODULE,LINK whose .clw opens with a bare MEMBER and a module-level MAP (so the BUILTINS.CLW intrinsics and the Win32 file calls resolve). All working state — buffers, the bit accumulators, the LZ77 hash chains, the Huffman decode tables — lives in class members, so the single global instance is reusable and the codec has no global side-effects.

The bit layout follows the RFC exactly: DEFLATE packs bits LSB-first, while Huffman codes are defined MSB-first, so the writer reverses each code's bits before packing and the reader decodes canonically bit-by-bit. Output grows a class-owned buffer; for back-reference copies the engine reserves space up front so the buffer never moves mid-copy (which also makes overlapping runs — the heart of run-length compression — fall out for free).

C fast-path (4× faster) #

The pure-Clarion codec is fast enough for most work, but for big files or high throughput there is an optional C engine — mc.c, our own DEFLATE port — that compiles with Clarion's own C compiler and makes compression about 4× faster, with no external DLL and no third-party library.

EngineCompress (4 MB, level 6)DecompressRatio
Pure Clarion (default)~844 ms~15 ms491 KB
C fast-path~200 ms~15 ms485 KB (full 32 KB window)

It is the same algorithm in C — a clean-room port of our Clarion Deflate, not miniz/zlib/StringTheory — so it produces byte-compatible output (the engines interoperate freely: compress with one, decompress with the other). C also has no Clarion 64 KB-array limit, so the matcher uses the full 32 KB DEFLATE window, giving a slightly better ratio as well.

Turning it on

  1. Copy mc.c to the Clarion redirection path, alongside CompressClass.inc / .clw.
  2. Edit one line near the top of CompressClass.inc: 
    CmpUseC            EQUATE(1)        ! 0 = pure Clarion (default), 1 = C fast-path
  3. Rebuild. CompressClass now routes Compress / Decompress (and the file methods) through the C engine via PRAGMA('compile(mc.c)'). Your code is unchanged — same class, same API.
How the switch works CmpUseC is a compile-time equate. When it is 0 (the default) the C prototypes, the PRAGMA, and the C dispatch are all OMITted — so a pure-Clarion build needs no mc.c at all and is completely unaffected. When it is 1, Clarion compiles mc.c in and the methods call it.
Single-threaded The C engine uses file-scope work tables, so it is single-threaded. If you compress from multiple threads concurrently, leave the C fast-path off and use the per-instance pure-Clarion engine.

The .NET oracle #

Clarion cannot be run from CI, so — as with QrCodeCore and BarcodeCore — a small .NET tool, designer/CompressCore/, is the independent reference. .NET's own GZipStream / ZLibStream / DeflateStream are the validated encoder/decoder, so there is nothing to re-implement; the tool closes the loop both ways:

  • INFLATE test — .NET compresses a corpus; the Clarion DecompressFile must reproduce the original bytes.
  • DEFLATE test — Clarion CompressFile writes a .gz; .NET decompresses it and confirms it matches.
# generate golden vectors (caseN.bin + caseN.bin.gz/.zz/.raw + manifest)
dotnet run --project designer/CompressCore -- vectors ./vectors
# …have the Clarion app DecompressFile each .bin.gz and CompressFile each .bin to .bin.cgz…
dotnet run --project designer/CompressCore -- verify  ./vectors    # checks the Clarion gzip output

The corpus deliberately spans the code paths: empty, single byte, long compressible text, long runs, binary with NUL bytes, incompressible random data, and an all-zero buffer.

Class reference #

Everything myCompress does lives in one class, CompressClass. This chapter is the full anatomy: how it's declared, the equates it ships, its lifecycle, and every public method with its parameters and return value.

Declaration

CompressClass CLASS,TYPE,MODULE('CompressClass.CLW'),LINK('CompressClass.CLW')
  • TYPE — a type, so you declare your own named instance (the template declares one global Zip). You could declare more for independent configurations.
  • MODULE/LINK — the engine compiles into its own object and links in automatically.
  • The .clw carries a bare MEMBER + module-level MAP (folds in BUILTINS.CLW and the kernel32 file calls). No external state — safe in multi-DLL apps and across threads.

Equates

EquateValueMeaning
Cmp:Raw0Raw DEFLATE — no header or trailer.
Cmp:Zlib1zlib wrapper — 2-byte header + Adler32.
Cmp:Gzip2gzip wrapper — 10-byte header + CRC32 + ISIZE.

Lifecycle

Construct runs automatically: it sets Level = 6, Format = Cmp:Gzip and builds the one-time tables (the CRC32 table, the length/distance base+extra tables, and the fixed-Huffman code table) via Init. Destruct frees the internal output buffer. Nothing else needs cleanup.

Public methods

Lengths are LONG; buffers are *STRING (passed by reference, never copied to bind). A negative return (or 0 from the file methods) means failure — read ErrText.

PrototypeParametersReturns / effect
Compress(*STRING pIn, LONG pInLen, *STRING pOut) pIn/pInLen — the bytes to pack and how many. pOut — output buffer, ≥ MaxCompressed(pInLen). Compresses into pOut in the current Format at Level. Returns the compressed byte count, or -1.
Decompress(*STRING pIn, LONG pInLen, *STRING pOut) pIn/pInLen — the compressed bytes. pOut — output buffer, big enough for the original. Auto-detects gzip/zlib/raw and inflates into pOut. Returns the original byte count, or -1.
MaxCompressed(LONG pInLen) pInLen — the input size. The safe number of bytes to allocate for a compress output buffer.
CompressFile(STRING pSrc, STRING pDst) pSrc/pDst — source and target paths. Compresses the whole file in the current Format. Returns 1 ok / 0 fail.
DecompressFile(STRING pSrc, STRING pDst) pSrc/pDst — source and target paths. Auto-detects the format and decompresses to pDst. Returns 1 / 0.
SelfTest() — Round-trips a known buffer through raw, zlib and gzip and checks the bytes return identical. Returns 1 on pass.

Troubleshooting #

SymptomCause / fix
Decompressed data has trailing spaces / wrong lengthYou relied on LEN() or a padded STRING. Always carry the exact length returned by Compress and pass it to Decompress.
Compress returns -1pOut is too small — allocate Zip.MaxCompressed(len).
"Unresolved external" on a built-inThe .clw must open with a bare MEMBER and a module-level MAP so BUILTINS.CLW resolves. Don't remove them.
Template won't registerRe-save the class files as ANSI, not UTF-8.
.gz won't open in 7-ZipMake sure you wrote in gzip format (Zip.Format = Cmp:Gzip, the default) and stored the file with the exact returned length.
Other tool's .gz won't decompress hereConfirm it is gzip/zlib/raw DEFLATE (not zip/7z/bzip2/zstd). myCompress reads the DEFLATE family only.
