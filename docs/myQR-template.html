myQR

A self-contained Clarion 12 ABC procedure extension that renders a QR code into an IMAGE control. The QR value can be a design-time literal or any variable/expression you change in code, and the image auto-refreshes when the value changes. This is the programmer's reference: prompts, generated code, the helper API, and the runtime facts behind it.

myQR · version 1.00 · FAMILY('ABC')

Overview ↑ top

Drop the extension on a window procedure, point it at a sized IMAGE control, and give it a value. At run time the window shows a QR code for that value.

  • Value in the template or in code — type a quoted literal for a fixed code, or a Clarion variable/expression to drive it from your logic.
  • Auto-refresh — a window timer watches the value and reloads the QR whenever it changes; or call DO myQRRefresh yourself.
  • No QR library — Clarion has no built-in QR encoder, so the PNG is fetched from a free public web service and loaded into the image. Self-contained: no external .inc/.clw.
Reads this first The QR value is sent over HTTPS to a third-party service (api.qrserver.com) on every render, and an internet connection is required. See Privacy & internet before encoding anything sensitive.

Install & register ↑ top

  1. Add an IMAGE control to the window (an Image — not a Region) and size it.
  2. IDE: Setup ▸ Template Registry ▸ Register → pick templates/myQR/myQR.tpl.
  3. Open the window procedure, click Extensions, add myQR - QR code into an image control.
  4. On the General tab pick the image control and set the value/size/ECC; Generate, build, run.
Per procedure myQR is a procedure extension (like myPie), so add one per window that needs a QR. Two QR images on one window each get their own temp file (keyed by the control field number), so they never clash.

The value: literal vs code ↑ top

%myQRValue is emitted verbatim into the generated source, so it is evaluated at run time. That gives you two modes from one prompt:

ModeType in the promptGenerated
Fixed (literal)'https://example.com' (with quotes) myQR:Cur = 'https://example.com'
Driven (from code)Cus:Email or loc:URL or 'ID:' & CLIP(Cus:Id) myQR:Cur = Cus:Email
Driving it Change your variable anywhere in the procedure; with auto-refresh on, the next timer tick notices and reloads. Or do it immediately after you set the value with DO myQRRefresh.

Rendering & auto-refresh ↑ top

  • On window open — the QR is rendered once; if auto-refresh is on, the window timer is started.
  • Auto-refresh — on each EVENT:Timer the value is re-read and compared to what was last rendered; the network is only hit when it actually changed.
  • Manual — DO myQRRefresh from any embed re-renders the current value immediately (works whether or not auto-refresh is on).
Window-timer caveat Auto-refresh sets 0{PROP:Timer}. If the procedure already uses the window timer for something else, turn auto-refresh off and call DO myQRRefresh yourself after you change the value.

How the image is made ↑ top

There is no QR drawing in Clarion. myQRLoad builds a request to the free goqr.me service and downloads the PNG:

https://api.qrserver.com/v1/create-qr-code/?size=SxS&margin=M&ecc=L|M|Q|H&data=<url-encoded value>

The download uses curl.exe (ships with Windows 10/11 at %SystemRoot%\System32\curl.exe), launched hidden (no console flash) and synchronously (we wait for it) so the file is on disk before the image loads:

  1. Clear the image's hold on the temp file: ?img{PROP:Text} = '', then REMOVE(tempfile).
  2. Build curl -s -L --max-time 15 -o "tempfile" "url" and run it with CreateProcessA (CREATE_NO_WINDOW + SW_HIDE), then WaitForSingleObject.
  3. Trust the file, not the exit code: if the temp PNG now exists and is non-empty (DIRECTORY() size > 0), load it with ?img{PROP:Text} = tempfile and return success.

The temp file is .\myQR_<feq>.png in the working directory, keyed by the image control's field number.

Embed points & generated code ↑ top

All guarded by WHERE(%myQRDisable=0 AND %myQRImage):

EmbedEmits
%GlobalMapShort-form prototypes for myQRUrlEncode and myQRLoad.
%ProgramProceduresThe two helper bodies (the kernel32 prototypes live in myQRLoad's own local MAP).
%DataSectionmyQR:Last, myQR:Cur (change detection), myQR:Ecc.
%ProcedureRoutinesThe myQRRefresh routine.
%WindowManagerMethodCodeSection,'TakeWindowEvent'
PRIORITY(2000)		 Open + timer handler (self-contained CASE EVENT(), never RETURNs).

For an image ?QR with value loc:URL, auto-refresh on, size 200, ECC Medium, margin 1, poll 50, the per-procedure code is:

myQRRefresh ROUTINE
  myQR:Ecc = 'M'                          ! ECC letter, resolved at generate time
  myQR:Cur = loc:URL                     ! the value expression, verbatim
  myQRLoad(?QR, myQR:Cur, 200, myQR:Ecc, 1)
  myQR:Last = myQR:Cur

  ! injected at the top of WindowManager.TakeWindowEvent
  CASE EVENT()
  OF EVENT:OpenWindow
    0{PROP:Timer} = 50                   ! start the value-watch poll
    DO myQRRefresh                       ! first render
  OF EVENT:Timer
    myQR:Cur = loc:URL
    IF myQR:Cur <> myQR:Last           ! only hit the network on a real change
      DO myQRRefresh
    END
  END

A fixed value (e.g. 'Hello') with auto-refresh off generates just the OF EVENT:OpenWindow / DO myQRRefresh arm.

Privacy & internet ↑ top

The value leaves the machine Every render sends the QR value over HTTPS to api.qrserver.com (a third party) and needs a live internet connection. Do not encode passwords, tokens, account numbers or any data you aren't willing to transmit to that service.

A small temp PNG (myQR_<feq>.png) is written to the working directory and loaded into the image; it is cleared and re-downloaded on each refresh. If that directory isn't writable, pick a writable location (see Customising).

Customising / going offline ↑ top

  • Your own endpoint — for a private/offline deployment, edit myQRLoad's loc:URL to point at an in-house QR service (same ?data= contract), or swap the download for a local QR library/DLL that writes a PNG.
  • Simpler download — if you don't mind a brief console flash, the template contains a commented one-line fallback using RUN('curl …', 1) instead of CreateProcessA.
  • Colours — goqr.me also accepts &color= and &bgcolor= (RRGGBB); add them to loc:URL if you want a coloured code.
  • Temp location — change loc:File from .\myQR_… to a writable path (e.g. under the user's profile) if the working directory is read-only.

Multi-DLL note ↑ top

The helper bodies are emitted into %ProgramProcedures (the program/EXE module), matching myPie/myFontChanger. This targets single-EXE apps. For a multi-DLL app, host the extension in the program (EXE) app, or move the helper prototypes/bodies to the shared/root DLL and export them so every DLL that uses myQR can call them.

Helper reference ↑ top

myQR is class-free: there is no external .inc/.clw to register. Instead the template drops a small set of helper procedures straight into the program module, prototyped in the global MAP and bodied in %ProgramProcedures. This chapter is the full anatomy — where each helper lives, its real prototype, every parameter, the return value, and what it does.

Where they live

The two function-style helpers are prototyped short-form in the global map and defined in the program (EXE) module; a third helper is a per-procedure ROUTINE:

! global MAP, emitted at %GlobalMap
myQRUrlEncode(STRING pText),STRING
myQRLoad(SIGNED pImageFeq, STRING pData, SIGNED pSize, STRING pEccLetter, SIGNED pMargin),BYTE
  MODULE('kernel32')                          ! Win32 launch APIs, GLOBAL map only
myQR_CreateProcess(...),LONG,PASCAL,PROC,NAME('CreateProcessA')
myQR_WaitObject(LONG,ULONG),LONG,PASCAL,NAME('WaitForSingleObject')
myQR_CloseHandle(LONG),LONG,PASCAL,PROC,NAME('CloseHandle')
  END
  • The bodies of myQRUrlEncode and myQRLoad are written long-form at column 1 into %ProgramProcedures (the program/EXE module's DATA region), an EXE-only embed — so they compile into the program, not a class object.
  • The kernel32 APIs are declared with unique Clarion labels bound to the real exports by NAME() (myQR_CreateProcess → CreateProcessA, etc.), so they never clash with any CloseHandle/WaitForSingleObject the ABC runtime already declares. They must live in the global map — a local MAP rejects the MODULE('dll') external declaration.
  • Every emission is guarded by WHERE(%myQRDisable=0 AND %myQRImage), so a disabled template contributes nothing.

Helper procedures

These are the procedures you can call from your own embed code. pImageFeq is a control field-equate (SIGNED); the QR value and the 1-letter ECC are STRINGs; size and margin are SIGNED pixel/module counts.

PrototypeParametersReturns / effect
myQRUrlEncode(STRING pText), STRING pText — the raw value to put in the URL query. Returns pText percent-encoded for a URL query: RFC 3986 unreserved characters (A–Z a–z 0–9 - _ . ~) pass through, a space becomes %20, and every other byte becomes %XX. Called for you by myQRLoad; exposed so you can reuse the same encoding.
myQRLoad(SIGNED pImageFeq, STRING pData, SIGNED pSize, STRING pEccLetter, SIGNED pMargin), BYTE pImageFeq — the IMAGE control to load into. pData — the value to encode. pSize — the QR edge length in pixels (request is SxS). pEccLetter — the error-correction level, one letter (L/M/Q/H). pMargin — the quiet-zone width in modules. The download + load entry point. Builds the api.qrserver.com request URL (URL-encoding pData via myQRUrlEncode), clears the image's hold on its per-FEQ temp PNG (pImageFeq{PROP:Text}=''), then downloads the PNG with curl launched hidden and synchronously (myQR_CreateProcess with CREATE_NO_WINDOW + SW_HIDE, then myQR_WaitObject blocks until it exits). It trusts the file, not the exit code: after DIRECTORY()/EXISTS() confirm the temp PNG is present and > 0 bytes, it loads it with pImageFeq{PROP:Text}=loc:File and returns 1. Returns 0 when the PNG is missing or empty (curl absent, offline, or the service is down).

The refresh routine

myQRRefresh is not a procedure — it is a local ROUTINE generated into each host procedure (at %ProcedureRoutines), so you invoke it with DO, not as a call:

InvocationWhat it does
DO myQRRefresh Sets myQR:Ecc to the ECC letter resolved from the prompt at generation time, reads the current value into myQR:Cur (a literal is auto-quoted; a variable/expression is emitted verbatim and read live), calls myQRLoad(…) to download and load, then records the value in myQR:Last for change detection. It runs automatically on window open and on a detected value change; DO myQRRefresh forces a redraw any time from your own embed.
Procedure vs routine Call myQRUrlEncode and myQRLoad like functions — they take parameters and return a value. myQRRefresh changes the procedure's own state (myQR:Cur/myQR:Last) and lives in that procedure's data scope, so it is invoked with DO myQRRefresh and takes no arguments — it always renders whatever the configured value currently is.
It reaches the internet myQRLoad shells out to curl and sends the QR value over HTTPS to the third-party service api.qrserver.com. The value leaves the machine on every render. Do not pass secrets; for a private or offline deployment, repoint the URL in myQRLoad at your own QR endpoint or a local QR library (see Customising / offline).

Troubleshooting ↑ top

SymptomLikely cause / fix
Template won't registerA literal % in an emitted line. The shipped template escapes them (%%); if you edit the URL-encoder, keep that.
Image stays blankNo internet, the service is down, or curl.exe is missing (older Windows) — install curl or repoint the helper. Confirm myQR_<feq>.png appears.
Nothing updates when the variable changesAuto-refresh is off, or the value is a literal. Turn it on, or call DO myQRRefresh after changing the value.
The window's own timer stopped workingAuto-refresh took 0{PROP:Timer}. Turn auto-refresh off and drive the QR with DO myQRRefresh.
Drawing doesn't appear / wrong controlThe control must be an IMAGE (not a Region), and selected in the Image control prompt.
A console window flashesYou switched to the RUN() fallback; switch back to the CreateProcessA path for a hidden launch.

Verify checklist ↑ top

Claude can't run AppGen — after generating, confirm:

  1. The .tpl registers without error.
  2. The window opens and shows a scannable QR for the value (scan it with a phone).
  3. Changing the bound variable updates the image within one poll interval (auto-refresh on), and DO myQRRefresh updates it immediately.
  4. A myQR_<feq>.png appears in the working directory and no console window flashes.
